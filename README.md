# Gemini Watermark Remover

A C++20 CLI tool for removing visible watermarks from images generated by Google Gemini.

| Watermark | Type | Where | Visibility | Status |
|-----------|------|-------|------------|--------|
| Gemini sparkle logo | Alpha-blended overlay | Gemini-generated images | Visible (semi-transparent) | Implemented |
| Veo text watermark | Alpha-blended text overlay | Veo-generated videos | Visible | Planned |

## Project Status

**Phases 1–2 complete.** Visible watermark removal via reverse alpha blending with NCC detection and inpainting. 17 tests passing, clean build.

## Quick Start

### Build

Requires: CMake 3.21+, vcpkg, Ninja, C++20 compiler.

```bash
cmake --preset mac-x64-Release
cmake --build build/mac-x64-Release
```

### Usage

```bash
# Auto-detect and remove visible watermarks
wmr remove input.png -o output.png

# Detect watermarks without modifying
wmr detect input.png

# Remove visible watermark only
wmr visible input.png -o output.png

# Batch process a directory
wmr remove input_dir/ -o output_dir/ --recursive
```

### Common Options

| Flag | Description |
|------|-------------|
| `-o, --output` | Output path (file or directory) |
| `-f, --force` | Skip detection, assume watermark present |
| `--force-small` | Force 48x48 watermark size |
| `--force-large` | Force 96x96 watermark size |
| `--inpaint-strength` | Inpaint strength 0.0–1.0 |
| `--recursive` | Process directories recursively |
| `-v, --verbose` | Verbose output |
| `-V, --version` | Show version |

## SynthID Note

SynthID invisible watermarks use a deep learning encoder-decoder architecture (arXiv:2510.09263). Our research showed that FFT-based correlation metrics sit at random baseline (~51-52%) for both watermarked and non-watermarked images — classical signal processing cannot detect or remove this watermark type. The `synthid/` source code and `synthid`/`build-codebook` CLI subcommands remain in the codebase for reference but are not actively maintained.

## Technical Background

### Visible Watermark Removal (Gemini & Veo)

Gemini and Veo apply visible watermarks using **alpha blending**:

```
watermarked = alpha * logo + (1 - alpha) * original
```

Removal reverses the blending equation:

```
original = (watermarked - alpha * logo) / (1 - alpha)
```

This is a deterministic, mathematically exact operation when the alpha map is known and the image hasn't been resized or recompressed. Residual artifacts from post-processing are cleaned up using inpainting (Navier-Stokes, TELEA, or soft Gaussian blend) and optionally AI-based denoising.

Gemini uses two watermark sizes:
- **48x48** for images <= 1024px on either dimension
- **96x96** for images > 1024px on both dimensions

Both are placed in the bottom-right corner.

## Source Projects

This project builds on the research and techniques from:

### [GeminiWatermarkTool](https://github.com/allenk/GeminiWatermarkTool) — by Allen Kuo (allenk)

Gemini visible watermark removal via reverse alpha blending (C++20).

- Reverse alpha blending with calibrated 48x48 and 96x96 alpha masks
- Three-stage NCC watermark detection (spatial, gradient, variance)
- AI-assisted residual cleanup (FDnCNN neural network via NCNN + Vulkan)
- Software inpainting fallback (NS, TELEA, Soft Inpaint)
- Also powers [VeoWatermarkRemover](https://github.com/allenk/VeoWatermarkRemover) for video watermark removal

### [reverse-SynthID](https://github.com/aloshdenny/reverse-SynthID) — by aloshdenny

SynthID invisible watermark detection and removal via spectral analysis (FFT-based approach, proven ineffective against SynthID's neural encoder).
